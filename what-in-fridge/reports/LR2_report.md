# Лабораторна робота 2: Створення структури проєкту (семантична, адаптивна верстка)

## 1. Тема проєкту
**«Що в холодильнику?»** — веб‑застосунок, який допомагає користувачеві швидко визначити, що можна приготувати з наявних інгредієнтів.

## 2. Мета роботи
1) Створити **структуру веб‑проєкту** (HTML/CSS/JS) з базовою навігацією.  
2) Реалізувати **повну семантичну та адаптивну верстку** з урахуванням **SEO** та **A11Y**.  
3) Підготувати матеріали для звіту: вигляд сторінок та протоколи валідації.

## 3. Глибокий аналіз вимог

### 3.1. Цільова аудиторія
- Студенти та працюючі люди, які готують вдома.
- Користувачі, що шукають «швидко і без стресу» — переважно **з мобільного**.

### 3.2. Основні сценарії (User Stories)
- **US1:** Як користувач, я хочу відмітити продукти, які є вдома, щоб отримати список страв.  
- **US2:** Як користувач, я хочу бачити % відповідності, час та складність, щоб швидко обрати рецепт.  
- **US3:** Як користувач, я хочу перейти на детальну сторінку рецепта та побачити кроки приготування.  
- **US4:** Як користувач, я хочу додавати рецепти в «Обране», щоб не шукати їх знову.

### 3.3. Функціональні вимоги
- Вибір інгредієнтів через чекбокси (збереження у **localStorage**).
- Формування списку рецептів з ранжуванням за відповідністю.
- Фільтри: режим відповідності (повна/часткова), максимальний час, пошук.
- Сторінка рецепта з відміткою наявних/відсутніх інгредієнтів.
- Додавання/видалення з «Обраного».

### 3.4. Нефункціональні вимоги
- **Адаптивність:** мобільні та десктопні екрани, responsive grid, sticky‑actions.
- **A11Y:** skip-link, семантика, фокус‑стани, aria‑атрибути, альтернативні тексти.
- **SEO:** title/description на кожній сторінці, canonical, sitemap/robots, JSON‑LD Recipe на сторінці рецепта.
- **Чистий код:** відокремлення стилів/скриптів, повторне використання компонентів, зрозуміла структура.

## 4. Навігація та IA (Information Architecture)
Сторінки:
- `index.html` — головна (CTA, переваги, короткий опис).
- `ingredients.html` — вибір продуктів.
- `recipes.html` — результати + фільтри.
- `recipe.html` — деталі рецепта.
- `about.html` — концепція та UX/SEO/A11Y обґрунтування.
- `contacts.html` — демо‑форма зворотного зв’язку.

## 5. Реалізація UX‑факторів (ключові рішення)
- **Швидкий старт:** CTA на головній веде одразу на вибір продуктів.
- **Візуальна ієрархія:** контрастні кнопки, великі заголовки, картки.
- **Зворотний зв’язок:** лічильник вибраних інгредієнтів, кількість знайдених рецептів.
- **Зниження помилок:** підказки на сторінці вибору продуктів, прості фільтри.

## 6. Структура проєкту
```
what-in-fridge/
  index.html
  ingredients.html
  recipes.html
  recipe.html
  about.html
  contacts.html
  css/
    style.css
    responsive.css
  js/
    script.js
  images/
    logo.svg
    icons.svg
    recipe-*.svg
  screenshots/
    *.png
  validation/
    README.txt
    *-validator.png
  tools/
    screenshot-playwright.*
    validate_w3c_steps.md
```

## 7. Валідація та скріни для звіту
- У папці `validation/` є **README.txt** з інструкцією, як швидко зробити скріншоти W3C Validator (File Upload).
- У папці `tools/` є скрипт для автоматичного створення **реальних** скріншотів сторінок через Playwright.

> Примітка: у ZIP вже є файли‑заглушки PNG з підписом PLACEHOLDER — їх можна замінити на реальні скріни після запуску скрипта або ручного скріншоту.

## 8. Висновок
Створено адаптивний, семантичний навчальний веб‑проєкт з базовою навігацією, SEO/A11Y‑складовою та інтерактивністю через JS (localStorage, фільтрація).
